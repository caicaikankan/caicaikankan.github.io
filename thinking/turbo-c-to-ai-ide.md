前几天刷到一篇文章，讲阿里云百炼 Coding Plan Lite 停售的事，心里咯噔一下。作者说，三个月前 8 块钱就能拿下的入门套餐，现在不仅涨价、缩额度，连续费都不让了。看着那些熟悉的名字 —— 火山方舟、智谱 GLM Coding Plan，突然就想起自己这十几年的编程路，从纯手写代码到 AI 一键生成，从为工具折腾到被工具解放，好像也不过是一场 "随手可得" 到 "物是人非" 的轮回。

我的编程启蒙，没有现在小孩那么多选择，更没有 AI 加持，纯粹是 "硬啃" 出来的。大学第一节 C 语言课，教材是谭浩强那本红皮书，配套工具是 Turbo C—— 现在提起来，估计很多年轻开发者都不知道这是什么。纯黑底白字的 DOS 界面，没有鼠标，全靠键盘快捷键操作，没有代码补全，没有语法高亮，甚至连复制粘贴都要靠记命令。那时候写代码，与其说是编程，不如说是 "练打字 + 背语法"，一个分号漏写、一个括号不匹配，就能让你对着屏幕查一下午。

现在回头想，谭浩强教材 + Turbo C 这套组合，简直是当年计算机系学生的 "劝退套餐"。教材重语法、轻原理，把编程变成了死记硬背的公式；工具落后到离谱，让你花 80% 的精力折腾环境，只剩 20% 的精力思考逻辑。我记得第一次写 Hello World，明明照着书敲完了代码，却怎么都运行不了，最后发现是忘记切换英文输入法，把英文分号打成了全角的 "；"。那种崩溃感，现在想起来还历历在目 —— 那时候的编程，从来不是 "解决问题"，而是 "先搞定工具"。

后来学校机房统一装了 VC++ 6.0，我才算真正走进了 "可视化编程" 的大门。虽然放到现在看，VC6 也是上古神器，但对比 Turbo C，简直是降维打击：有了窗口和鼠标，有了语法高亮，能建项目管理多文件，还能断点调试。我用它从 C 语言过渡到 C++，一用就是好几年，从本科用到读研，它陪着我写过无数个控制台程序，啃过函数重载、继承、虚函数的硬骨头，也陪着我熬过无数个排查中文符号报错的深夜。

读研期间，我开始接触 VS Code，算是从老旧 IDE 向现代编辑器的过渡。那时候 VS Code 还没有现在这么完善，但已经能感受到现代工具的便捷 —— 可自定义插件，支持多语言，界面简洁，比 VC6 灵活太多。只是那时候还没意识到，这只是工具迭代的开始，未来的编程世界，会发生翻天覆地的变化。

真正的转折点，是出国之后。到了国外学校，我才发现，编程原来可以不用这么 "折腾"。课程作业全用 MATLAB，科研则慢慢转向 Python，那种从 "手动开荒" 到 "解放生产力" 的反差，至今记忆犹新。MATLAB 简直是工科生的福音，矩阵运算天生支持，绘图几行代码就能出论文级别的图，官方工具箱覆盖各个学科，不用自己写底层算法，不用管内存分配，精力终于能放回科研本身 —— 那一刻我才明白，编程的本质，从来不是折腾语法和工具，而是用代码解决问题。

随着机器学习兴起，Python 逐渐取代 MATLAB，成为我科研的主力语言。而 PyCharm 的流行，更是给 Python 配上了一套堪比 MATLAB 的友好界面，智能补全、语法检查、一键调试，让写 Python 代码变成了一种享受。后来上网课用 Jupyter Notebook，边写边运行，极易复刻课程代码；做深度学习时用 Colab，蹭免费 GPU，不用自己搭环境、买显卡，在 GPU 还是稀缺资源的年代，简直是 "神器" 一般的存在。

这一路走下来，我几乎体验了除 Java 之外的所有主流编程语言，碰过的 IDE 更是从 Turbo C、VC6.0、VS Code，到 PyCharm、Jupyter、Colab，再到现在的 Claude Code、Trae 这类 AI 原生 IDE。工具越来越智能，门槛越来越低，从 "记语法、写代码、调 bug"，变成 "说需求、看结果、微调整"，编程彻底从 "手艺活" 变成了 "生产力工具"。

但越是亲历这种迭代，我心里的焦虑就越重 —— 现在让小朋友学编程，真的对吗？

我自己就是最好的例子：当年死磕的 Turbo C、VC6.0，现在早就进了博物馆；当年背得滚瓜烂熟的 C++ 语法、头文件配置，现在 AI 一键就能生成；当年花大量时间学的 MATLAB 函数，现在用 Python 几个库就能替代。我花了十几年时间，从落后的工具一步步追上潮流，可回头看，当年学的很多东西，都已经彻底落伍了，就像现在的小朋友学 Scratch、Python，十几年后，这些东西大概率也会被更先进的工具和模式替代。

这让我想起最近 AI 赛道的变化，和我亲历的编程工具迭代，其实是同一个逻辑。三个月前，阿里云百炼、火山方舟的 Coding Plan 还只要 8 块、9 块 9 一个月，厂家抢着拉新，客服打电话求着续费；三个月后，阿里云停售 Lite 套餐，智谱连续涨价，新模型只对高价 Pro 用户开放。不是厂家突然变坏了，而是 AI 已经从 "消费品" 变成了 "黄金"—— 当它真正成为生产力，能帮人赚钱的时候，价格就不再由成本决定，而是由它能创造的价值决定。

编程工具的迭代，其实也是一场 "黄金化" 的过程。从 Turbo C 到 AI IDE，工具越来越强大，越来越便捷，可入门门槛看似降低，实则在悄悄抬高。就像现在的新人，已经错过了 8 块钱就能体验 AI Coding 的时代，想要入门，成本比我们当年高出好几倍；就像当年的我，错过了更早期的编程工具红利，只能硬啃最落后的工具。

回到那个问题：现在让小朋友学编程，真的对吗？

我的答案是，对，但也不对。

不对的是，现在很多教育机构教的，依然是 "马上会过时" 的东西 —— 死背语法、记命令、刷固定题型，就像当年我们学 Turbo C 一样，把精力都花在了无关紧要的细节上。等这些小朋友长大，现在学的 Python、Scratch，大概率会像我们当年学的 Turbo C 一样，被淘汰出局，到时候他们还是要重新开始。

对的是，学编程这件事本身，从来不是为了学某一种语言、某一个工具，而是为了练一种思维 —— 把模糊的问题拆解成可执行的步骤，用逻辑解决问题，排查错误、优化方案。我当年学的 Turbo C、VC6.0 虽然过时了，但那些在痛苦中磨出来的逻辑思维、抽象能力、问题排查能力，却一直陪着我，让我能快速上手 MATLAB、Python、AI IDE，能快速适应工具的迭代。

这就像 AI Coding 的趋势一样，未来，大部分人不需要精通编程语言，不需要记住语法细节，只要会向机器提需求，能判断结果对错，就足够了。但这种 "提需求" 的能力，本质上还是编程思维 —— 能把需求说清楚，能拆解问题，能指出逻辑漏洞，而这些能力，恰恰是现在很多编程教育忽略的。

我常常想，如果当年有人告诉我，我花那么多时间学的 Turbo C、VC6.0，以后都会过时，我会不会少走一些弯路？但现在回头看，那些弯路，其实也不是白走的。正是那些在 Turbo C 里查分号的下午，在 VC6 里排查中文符号的深夜，在 MATLAB 里调试模型的凌晨，让我真正理解了编程的本质，也让我能从容应对每一次工具迭代。

现在的 AI IDE 越来越强大，大模型越来越聪明，编程的门槛越来越低，但这并不意味着我们可以不用学习、不用思考。就像阿里云百炼 Coding Plan 停售告诉我们的，那些 "随手可得" 的红利，从来不会一直存在；那些看似落后的工具和经历，也从来不是毫无意义。

从 Turbo C 到 AI IDE，我亲历了编程工具的黄金时代，也见证了行业的迭代与变迁。我知道，再过几年，现在的 AI 工具也会过时，会有更先进的技术出现，就像当年的 Turbo C 被 VC6 替代，VC6 被 VS Code 替代一样。但我不再焦虑，因为我明白，真正有价值的，从来不是某一种语言、某一个工具，而是我们在学习和实践中，沉淀下来的思维和能力。

至于现在的小朋友，与其让他们死背语法、练打字，不如让他们多思考、多尝试，培养解决问题的能力。毕竟，工具会过时，语法会淘汰，但思维能力，会陪伴他们一生。

工具的迭代从不停歇，红利的窗口终会关闭，就像当年的 Turbo C、如今的低价 Coding Plan，唯有沉淀在骨子里的思维与学习力，能让我们在每一次时代洗牌中，不被浪潮淘汰 —— 这或许，就是编程教会我们最珍贵的东西。
